# MechaGo — Índice de Tasks e Modelo de Trabalho

> **Modelo**: A IA da IDE gera tasks baseada nos marcos de conclusão do Roadmap.
> Não existe número fixo de tasks. Cada task cobre o que falta para completar a fase atual.
>
> **Fluxo**: Consultar Roadmap (marcos) → Gerar task → IA executa → Validar checklist → Repetir até marco completo

> **Nota de baseline (ja aplicada)**:
> - `RULES.md` e `MechaGo_Technical_Reference.md` foram alinhados ao repo real.
> - O repo real e os `package.json` dos workspaces sao a fonte da verdade imediata.
> - `apps/cliente` e `apps/pro` nao podem importar codigo-fonte de `apps/api/src`.
> - Contratos compartilhados ja extraidos para `packages/shared` devem ser preservados, nao recriados dentro das apps.
> - Ao gerar novas tasks, registrar apenas escopo funcional novo; limpezas tecnicas ja aplicadas entram como baseline, nao como feature.

---

## COMO FUNCIONA

```
1. A IA lê o Roadmap Completo → seção "MARCOS DE CONCLUSÃO"
2. Verifica o que JÁ está implementado no projeto
3. Identifica o que FALTA para o marco da fase atual
4. Gera a próxima task cobrindo o próximo bloco de funcionalidade
5. Implementa (backend + frontend juntos, fiel ao Stitch)
6. Valida com o checklist da task
7. Volta ao passo 2 até o marco estar 100% completo
```
