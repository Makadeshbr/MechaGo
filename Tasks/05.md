## BACKEND

### 1. Modulo Professionals (padrao 4 arquivos)

**Referencia**: Technical Reference secao 3.1 e 6.3

Criar `apps/api/src/modules/professionals/`:

```
professionals/
├── professionals.schemas.ts    # Zod schemas (register, update, online/offline)
├── professionals.routes.ts     # Rotas OpenAPI
├── professionals.service.ts    # Logica de negocio
├── professionals.repository.ts # Queries Drizzle
└── professionals.service.test.ts # Testes TDD
```

**Endpoints** (Technical Reference secao 6.3):

```
POST   /api/v1/professionals/register     # Cadastro (4 etapas em 1 request)
GET    /api/v1/professionals/me            # Perfil do profissional logado
PATCH  /api/v1/professionals/me            # Atualizar perfil
POST   /api/v1/professionals/me/online     # Ficar online (atualiza location)
POST   /api/v1/professionals/me/offline    # Ficar offline
PATCH  /api/v1/professionals/me/location   # Atualizar GPS (background)
GET    /api/v1/professionals/me/stats      # Estatisticas (ganhos, nota, aceite)
```

**Schemas Zod** (criar em professionals.schemas.ts):

- `registerProfessionalSchema`: name, cpf, phone, type (mecanico_movel, eletricista, borracheiro, guincho), vehicleTypesServed[], specialties[], maxRadiusKm, scheduleType
- `updateProfessionalSchema`: .partial() do register
- `goOnlineSchema`: latitude, longitude
- `updateLocationSchema`: latitude, longitude

**Regras de negocio**:

- CPF unico (validar formato + unicidade)
- vehicleTypesServed: array de vehicle_type enum (car, moto, suv, truck)
- maxRadiusKm: minimo 3, maximo 50
- scheduleType: "daytime" | "nighttime" | "24h"
- Ao ficar online: persistir latitude/longitude decimais no schema atual e usar PostGIS no repository para queries geoespaciais
- Ao ficar offline: setar isOnline = false
