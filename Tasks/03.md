## BACKEND (Módulos Users & Vehicles)

**1. Módulo Users (Básico para o fluxo)**
- **Arquivos:** `users.schemas.ts`, `users.repository.ts`, `users.service.ts`, `users.routes.ts`
- **Endpoints:** 
  - `GET /api/v1/users/me` (retorna os dados do usuário logado via `c.get('userId')`)
  - `PATCH /api/v1/users/me` (atualiza dados básicos do perfil)
- **Testes (TDD):** Criar `users.service.test.ts`. Validar que o `passwordHash` NUNCA é retornado no objeto de resposta e que o acesso de usuário inexistente (ou token órfão) é tratado.

**2. Módulo Vehicles**
- **Arquivos:** `vehicles.schemas.ts`, `vehicles.repository.ts`, `vehicles.service.ts`, `vehicles.routes.ts`
- **Endpoints:**
  - `POST /api/v1/vehicles` (cadastra)
  - `GET /api/v1/vehicles` (lista os veículos do usuário logado)
  - `PATCH /api/v1/vehicles/:id`
  - `DELETE /api/v1/vehicles/:id`
- **Regras de Negócio:** 
  - Validar placa (formato antigo AAA-1234 ou Mercosul AAA1A23).
  - Máximo de 5 veículos por cliente.
  - Placa não pode ser duplicada no sistema.
- **Testes (TDD):** Criar `vehicles.service.test.ts` testando os limites e duplicações.
